# 📦 Проект: тестирование API Яндекс.Прилавка (новая функциональность)

## 📋 О проекте
Тестирование новой функциональности бэкенда Яндекс.Прилавка: работа с наборами, курьерской доставкой и корзиной. Проверка реализована через Postman, тестовая документация оформлена в виде чек‑листа, дефекты фиксировались в Google Таблицах.

## 🎯 Основные задачи

### 1. Анализ требований и подготовка тестовой документации
- Изучена документация API в Apidoc и требования к бэкенду.
- Спроектирован чек‑лист для покрытия новой функциональности (без проверки авторизации).
- Чек‑лист размещён в Google Таблице с доступом на комментирование.  
  🔗 [Чек‑лист тестирования API](https://docs.google.com/spreadsheets/d/1i-oC4gzX_kWMsZjystlZrJoVMnvA1_1d9Jqs0BtSMww/edit?gid=2083771975#gid=2083771975)

### 2. Тестирование API через Postman
- Протестированы эндпоинты:
  - `POST /api/v1/kits/{id}/products` — добавление продуктов в набор.
  - `POST /fast-delivery/v3.1.1/calculate-delivery.xml` — расчёт стоимости доставки курьерской службой «Привезём быстро».
  - `GET /api/v1/orders/:id` — получение списка продуктов в корзине.
  - `PUT /api/v1/orders/:id` — добавление продуктов в корзину.
  - `DELETE /api/v1/orders/:id` — удаление корзины.
- Проверки включали позитивные и негативные сценарии, граничные условия и бизнес‑логику.

### 3. Фиксация дефектов
- При обнаружении ошибок дефекты фиксировались в Google Таблицах с чёткими шагами воспроизведения и ожидаемым поведением.

## 🧰 Применяемые техники и инструменты
- Техники: анализ требований, проектирование чек‑листов, проверка граничных значений и негативных сценариев.
- Инструменты: Postman (тестирование API), Google Таблицы (чек‑листы и баг‑репорты).
- Предусловия: тестовый стенд — `https://{id}.serverhub.praktikum-services.ru/`, документация API — `https://{id}.serverhub.praktikum-services.ru/docs/`.
