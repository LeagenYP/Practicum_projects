# Моё портфолио проектов по тестированию ПО

Здесь собраны проекты, в которых я отрабатывал навыки ручного и автоматизированного тестирования. Подробнее о каждом — в README внутри соответствующей папки.

---

## 🗺️ Яндекс.Маршруты (ручное тестирование)
Тестирование заказа каршеринга: кроссбраузерные проверки, сверка с макетами Figma, тест‑кейсы и чек‑листы (классы эквивалентности, граничные значения). Дефекты фиксировал в Google Таблицах.

## 🚇 Яндекс.Метро (ручное тестирование)
Тестирование Android‑приложения: функциональное и нефункциональное тестирование в Android Studio, исследовательское тестирование, чек‑листы. Баг‑репорты с приоритетами — в Google Таблицах.

## 🛒 Яндекс.Прилавок (ручное тестирование)
Тестирование API и бэкенда: проверки в Postman, акцент на бизнес‑логике и граничных условиях. Выполнял смоук, компонентное, модульное и регрессионное тестирование.

## 🛵 Яндекс Самокат (проектирование и ручное тестирование)  
Проектирование и ручное тестирование веб‑ и мобильного приложения: чек‑листы и валидация полей для экрана «Статус заказа» и «Сделать заказ», тест‑кейсы по макетам Figma и требованиям (в т. ч. по вёрстке и функциональности). Баг‑репорты с приоритетами и статусами (PASSED/FAILED) — в Google Таблицах.

## 🍔 Stellar Burgers (автоматизированное тестирование)
Комплексная автоматизация: юнит‑тесты (JUnit 4, Mockito, JaCoCo), API‑автотесты (REST Assured), UI‑автотесты (Selenium WebDriver). Отчётность — Allure.

## 🛴 Яндекс.Самокат (автоматизированное тестирование)
Автотесты UI (Selenium WebDriver, кроссбраузерно) и API (REST Assured). Результаты и разбор ошибок — через отчёты Allure.
