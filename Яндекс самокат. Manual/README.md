# 🛴 Проект: ручное тестирование сайта «Яндекс Самокат» и проектирование тестов для Android‑приложения

## 📋 О проекте
Проект включает два блока работ: ручное тестирование веб‑приложения «Яндекс Самокат» (экран «Статус заказа», валидация полей на экране «Сделать заказ» и прочая функциональность) и проектирование тест‑кейсов для мобильного Android‑приложения. Все результаты, чек‑листы, данные валидации и баг‑репорты зафиксированы в Google Таблицах.

## 🎯 Основные задачи

### 1. Проектирование и тестирование веб‑приложения
- Изучены требования к веб‑приложению и макеты в Figma (лендинг не тестировался).
- Составлен чек‑лист по экрану «Статус заказа» — размещён на вкладке «Задание 1: чек‑лист».
- Для экрана «Сделать заказ» подготовлены проверки валидации полей в виде таблицы по шаблону — на вкладке «Задание 1: данные валидации».
- Проведено тестирование функциональности по чек‑листам, требованиям и макетам; каждый тест отмечен как PASSED или FAILED.
- Баги, не покрытые тестовой документацией, зафиксированы на вкладке «Задание 1: баги вне тестовой документации».
- Для всех FAILED‑тестов заведены баг‑репорты на вкладке «Баг‑репорты» с указанием ID в таблице результатов.

### 2. Проектирование тестов для мобильного приложения (Android)
- Изучены требования к мобильному приложению, особое внимание уделено функциональности, выделенной жирным шрифтом.
- Спроектированы тест‑кейсы на указанную функциональность и на вёрстку по макетам Figma.
- Тест‑кейсы размещены на вкладке «Задание 2: тест‑кейсы».

## 🧰 Применяемые техники и инструменты
- Техники: чек‑листы, тест‑кейсы, валидация полей, анализ требований и макетов, ручное тестирование.
- Инструменты: Figma (макеты), Google Таблицы (чек‑листы, таблицы валидации, тест‑кейсы, баг‑репорты).
- Документация: требования к веб‑ и мобильному приложению, инструкция по тестированию статусов заказа.
