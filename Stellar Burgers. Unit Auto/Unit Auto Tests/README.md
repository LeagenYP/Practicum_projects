# 🧪 Проект: юнит‑тесты для приложения Stellar Burgers

## 📋 О проекте
Проект посвящён написанию юнит‑тестов для класса `Burger` в космической бургерной Stellar Burgers. Тесты покрывают логику сборки бургера: установку булочек, добавление/удаление/перемещение ингредиентов, расчёт цены и формирование чека. Для изоляции зависимостей использованы моки (Mockito), для вариативности проверок — параметризация, покрытие кода — 100 % (контролируется JaCoCo). Проект — Maven‑модуль на Java 11 с JUnit 4.

---

## 🧪 Реализованные тесты

### 🔹 TestBurger — базовые операции и расчёт цены
- `testBurgerSetBuns` — проверка установки булочек: убеждаемся, что поле `bun` корректно принимает переданный мок‑объект `Bun`.
- `testBurgerAddIngredient` — проверка добавления ингредиента: валидируем, что после вызова `addIngredient` размер списка `ingredients` увеличивается на 1.
- `testBurgerRemoveIngredients` — проверка удаления ингредиента по индексу: после `removeIngredient(0)` список ингредиентов становится пустым.
- `testBurgerMoveIngredients` — проверка перемещения ингредиентов: подтверждаем, что элементы списка корректно меняют позиции (например, перемещение второго ингредиента на первую позицию).
- `testBurgerGetPrice` — проверка расчёта итоговой цены: с помощью моков задаём цены булочки и ингредиента, проверяем, что `getPrice()` возвращает корректную сумму (цена двух булочек плюс сумма ингредиентов) с допустимой погрешностью `0.01`.

### 🔹 TestBurgerReceipt — параметризованные тесты формирования чека
- `testBurgerGetReceipt` — проверка формата чека `getReceipt()` на трёх наборах данных:
  - бургер с одним ингредиентом (сыр): проверяем структуру строк и итоговую цену;
  - бургер без ингредиентов: проверяем отсутствие строк с ингредиентами и корректный расчёт цены только по булочкам;
  - бургер с двумя ингредиентами (сыр и помидор): проверяем порядок строк, наличие всех ингредиентов и итоговую сумму.
  Каждый прогон использует отдельный набор входных данных (булочка, список ингредиентов, ожидаемая строка чека) через параметризацию.
