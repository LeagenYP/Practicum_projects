# 🚇 Проект: тестирование мобильного приложения «Яндекс Метро» после рефакторинга

## 📋 О проекте
Проект посвящён тестированию мобильного приложения «Яндекс Метро» (Android) после рефакторинга: проверке изменённой функциональности и регрессионному тестированию перед релизом. По итогам работ подготовлен отчёт.

## 🎯 Основные задачи

### 1. Подготовка к функциональному тестированию
- Проанализированы требования (с выделением изменённых участков), при необходимости использована mindmap для структурирования логики.
- Составлен чек‑лист проверок по затронутым требованиям.

### 2. Подготовка к регрессионному тестированию
- Сформирован чек‑лист регрессионных проверок с учётом специфики мобильного приложения.
- Включены проверки мобильных сценариев:
  - работа при смене типа сети (3G, LTE, Wi‑Fi), восстановлении связи, в авиарежиме;
  - поведение при уведомлениях, входящих звонках, переключении между приложениями;
  - ориентация экрана, масштабирование интерфейса;
  - геолокация, обновление приложения, режим энергосбережения, низкий заряд батареи и др.

### 3. Выполнение тестирования
- Тестирование проводилось на конфигурации: Honor 8 (Android 9.0 Pie, 1080×1920, 5.5″) — на эмуляторе Android Studio либо на физическом устройстве.
- Проверки выполнены по подготовленным чек‑листам.
- Выявленные дефекты зафиксированы в Google Таблицах.

### 4. Отчёт о тестировании
- Подготовлен итоговый отчёт с описанием выполненных проверок, статусами и перечнем дефектов.

## 🧰 Применяемые техники и инструменты
- Техники: функциональное и регрессионное тестирование, анализ требований, структурирование через mindmap.
- Инструменты: эмулятор Android Studio, Google Таблицы (для баг‑репортов и отчётности).
- Документация: чек‑листы, отчёт о тестировании, баг‑репорты.
