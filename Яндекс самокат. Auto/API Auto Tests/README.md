# 🧪 Проект: API‑тестирование учебного сервиса «Яндекс Самокат»

## 📋 О проекте
Проект посвящён автоматизированному тестированию API сервиса «Яндекс Самокат» (qa-scooter.praktikum-services.ru/docs/). Реализованы тесты на RestAssured для ключевых сценариев: создание и авторизация курьеров, создание заказов с вариациями параметров, получение списка заказов. Применены параметризация, Allure‑отчётность, очистка тестовых данных после каждого теста. Проект — Maven‑модуль на Java 11 с JUnit 4.

---

## 🧪 Реализованные тесты

### 🔹 TestCreateCourier — тесты создания курьера
- `courierCreateSuccessTest` — проверка успешного создания курьера: валидация статуса `201 Created` и поля `ok: true`.
- `cannotCreateSameCourierTwiceTest` — проверка запрета на дублирование логина: статус `409 Conflict` и сообщение «Этот логин уже используется» (отмечен баг с расхождением текста ошибки).
- `cannotCreateCourierWithoutLogin` — валидация отсутствия логина: статус `400 Bad Request` и сообщение «Недостаточно данных для создания учётной записи».
- `cannotCreateCourierWithoutPassword` — валидация отсутствия пароля: статус `400 Bad Request` и аналогичное сообщение об ошибке.
- `cannotCreateCourierWithoutFirstName` — проверка допустимости создания курьера без имени: статус `201 Created`.
- `clearData` (`@After`) — очистка тестовых данных: удаление созданного курьера через авторизацию и вызов удаления (с обработкой случая, когда курьер не был создан).

### 🔹 TestCreateOrder — тесты создания заказа (с параметризацией)
- `createOrderTest` — создание заказа с разными вариантами цветов: пустой массив, `BLACK`, `GREY`, комбинация `BLACK, GREY`. Проверки: статус `201 Created`, наличие поля `track` в ответе.
- `cancelCreatedOrder` (`@After`) — отмена созданного заказа по значению `track` из ответа, чтобы сохранить чистоту данных между тестами.

### 🔹 TestGetOrdersList — тест получения списка заказов
- `getOrdersListTest` — проверка, что эндпоинт возвращает список заказов и в ответе присутствует поле `orders` (не `null`).

### 🔹 TestLogInCourier — тесты авторизации курьера
- `successLogInTest` — успешная авторизация: статус `200 OK` и наличие поля `id` в ответе.
- `failLogInWithoutLoginTest` — ошибка при отсутствии логина: статус `400 Bad Request`, сообщение «Недостаточно данных для входа».
- `failLogInWithoutPasswordTest` — ошибка при отсутствии пароля: статус `400 Bad Request`, аналогичное сообщение (отмечен баг, требующий таймаута 15 сек).
- `failLogInWithWrongPasswordTest` — ошибка при неверном пароле: статус `404 Not Found`, сообщение «Учётная запись не найдена».
- `failLogInWithWrongLoginTest` — ошибка при неверном логине: статус `404 Not Found`, аналогичное сообщение.
- `prepareCourier` (`@Before`) — подготовка тестовых данных: создание курьера перед каждым тестом.
- `clearData` (`@After`) — удаление тестового курьера после завершения теста (с обработкой исключений).
